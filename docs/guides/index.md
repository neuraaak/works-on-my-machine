## Code Standards

WOMM follows these standards:

- **Python**: PEP 8, Black formatting, type hints
- **JavaScript**: ESLint, Prettier formatting
- **Testing**: pytest (Python), Jest (JavaScript)
- **Documentation**: Google-style docstrings, Markdown

### Version Control

- Use `.gitignore` (auto-generated by WOMM)
- Commit configuration files
- Exclude virtual environments and dependencies
- Use meaningful commit messages

### Code Quality

- Run `womm lint` before commits
- Use `womm spell` for documentation
- Run tests regularly (`pytest` or `npm test`)
- Format code automatically (Black, Prettier)
